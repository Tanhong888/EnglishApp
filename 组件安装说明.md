# EnglishAPP 组件安装说明

## 1. 文档目的

本文档用于说明 EnglishAPP 项目在 Windows 环境下的组件安装与启动步骤，覆盖：

- 后端服务（FastAPI）
- Windows 客户端（Flutter）
- 可选基础设施（PostgreSQL / Redis / Docker）

适用目录：`D:\AI\EnglishAPP`

---

## 2. 环境要求

### 2.1 操作系统

- Windows 10/11（推荐 64 位）

### 2.2 必装工具

- Git（建议最新版）
- Python `3.11+`
- Flutter SDK（支持 Windows desktop，建议 `3.22+`）

### 2.3 可选工具

- Docker Desktop（用于一键启动 PostgreSQL/Redis）

---

## 3. 项目目录结构

```text
EnglishAPP/
  backend/                    # FastAPI 后端
  desktop_app/                # Flutter Windows 客户端
  docker-compose.yml          # PostgreSQL + Redis（可选）
  .env.example                # 根环境变量示例
  backend/.env.example        # 后端环境变量示例
```

---

## 4. 后端组件安装（FastAPI）

### 4.1 安装依赖

```powershell
cd D:\AI\EnglishAPP\backend
python -m pip install -e .[dev]
```

### 4.2 核心依赖（由 `pyproject.toml` 管理）

- `fastapi`
- `uvicorn[standard]`
- `sqlalchemy`
- `alembic`
- `PyJWT`
- `passlib[argon2]`
- `pydantic` / `pydantic-settings`

测试依赖：

- `pytest`
- `httpx`

### 4.3 环境变量

复制示例文件并按需修改：

```powershell
cd D:\AI\EnglishAPP
Copy-Item .env.example .env
cd backend
Copy-Item .env.example .env
```

默认可直接使用 SQLite 快速启动。

### 4.4 数据库迁移

```powershell
cd D:\AI\EnglishAPP\backend
alembic -c alembic.ini upgrade head
```

### 4.5 启动后端

```powershell
cd D:\AI\EnglishAPP\backend
uvicorn app.main:app --reload --port 8000
```

访问地址：

- OpenAPI 文档：`http://127.0.0.1:8000/docs`
- API 根前缀：`http://127.0.0.1:8000/api/v1`

---

## 5. Windows 客户端组件安装（Flutter）

### 5.1 启用 Windows Desktop

```powershell
flutter config --enable-windows-desktop
flutter doctor
```

### 5.2 安装依赖

```powershell
cd D:\AI\EnglishAPP\desktop_app
flutter pub get
```

当前关键依赖：

- `flutter_riverpod`
- `go_router`
- `google_fonts`
- `shared_preferences`

### 5.3 启动客户端

```powershell
cd D:\AI\EnglishAPP\desktop_app
flutter run -d windows
```

---

## 6. 可选基础设施安装（PostgreSQL / Redis）

项目提供 `docker-compose.yml`：

```powershell
cd D:\AI\EnglishAPP
docker compose up -d
```

默认端口：

- PostgreSQL: `5432`
- Redis: `6379`

如果切换到 PostgreSQL，请同步修改 `DATABASE_URL`。

## 8. 常见问题

1. `ModuleNotFoundError: fastapi`
- 原因：使用了错误的 Python 解释器（如 Anaconda 环境）。
- 处理：确认 `python -m pip install -e .[dev]` 与 `python -m pytest` 使用同一个解释器。

2. Flutter 无法运行 Windows 桌面
- 处理：执行 `flutter config --enable-windows-desktop`，并通过 `flutter doctor` 修复依赖。

3. 端口冲突
- 处理：修改启动端口或停止占用进程。

4. 切换 PostgreSQL 后启动失败
- 处理：检查 `DATABASE_URL`、容器状态、并重新执行 Alembic 迁移。

---

## 9. 推荐安装顺序

1. 安装 Python + Flutter + Git
2. 安装后端依赖并启动后端
3. 安装 Flutter 依赖并启动 Windows 客户端
4. 再按需启用 Docker 的 PostgreSQL/Redis
