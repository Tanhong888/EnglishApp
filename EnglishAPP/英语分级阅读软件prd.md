# 英语分级阅读软件 PRD

## 1. 文档信息

**产品名称**：暂定《英阅通》  
**文档名称**：英语分级阅读软件产品需求文档（PRD）  
**版本号**：V1.0  
**面向用户**：18–26 岁大学生，核心面向英语四六级、考研英语备考人群  
**文档目的**：明确产品定位、目标用户、核心功能、页面结构、业务流程、数据指标与 MVP 范围，指导设计、开发、测试与后续迭代。

## 2. 产品背景

大学生在英语四六级与考研备考过程中，普遍面临以下问题：

1. **阅读输入不足**：日常英语输入内容单一，主要局限于词汇书、真题和教材，知识面较窄，阅读兴趣不足。
2. **生词负担重**：用户在阅读时频繁遇到陌生词汇，查词成本高，影响阅读连贯性。
3. **长难句理解困难**：面对复杂句式、从句、插入语、非谓语结构时，理解效率低。
4. **阅读与积累脱节**：读完文章后缺乏有效沉淀，生词、句型和理解能力难以持续提升。
5. **备考场景缺少分级内容**：市面上很多英语阅读产品要么偏泛阅读，要么偏刷题，缺少贴近考试场景且难度合理的分级精读工具。

因此，需要设计一款围绕“分级阅读 + 释义解析 + 阅读检测 + 词汇沉淀”的英语学习软件，帮助目标用户在备考过程中提升英语阅读能力、词汇积累效率与长难句理解能力。

## 3. 产品定位

### 3.1 产品定位

一款面向大学生四六级与考研备考场景的英语分级阅读软件，通过文章朗读、点词释义、句子解析、阅读小测与生词本功能，帮助用户提升阅读理解、词汇积累和长难句分析能力。

### 3.2 核心价值

为用户提供“**可读懂、可学习、可积累**”的英语阅读体验：

- **可读懂**：通过分级内容与句子解析降低阅读门槛
- **可学习**：通过点词释义与朗读辅助完成即时学习
- **可积累**：通过小测与生词本把阅读结果沉淀下来

### 3.3 产品形态

V1.0 先交付 **Windows 桌面 App**（面向备考用户的学习主链路），后续再扩展 Android 端与管理后台。

## 4. 产品目标

### 4.1 业务目标

1. 建立一个面向四六级/考研备考用户的英语阅读学习产品。
2. 通过高质量分级阅读内容与学习工具，提高用户留存与学习频次。
3. 建立“阅读—理解—检测—积累”的学习闭环，为后续扩展课程、题库、会员等能力打基础。

### 4.2 用户目标

1. 能根据自身英语水平快速找到合适难度的文章。
2. 在阅读过程中便捷完成查词、听读与句子理解。
3. 通过小测检验自己是否真正读懂文章。
4. 能持续积累生词与复习阅读中遇到的重要内容。

### 4.3 MVP 目标

在 V1.0（Windows 桌面版）完成以下最小可用闭环：

- 分级文章列表
- 文章朗读
- 点词释义
- 重点句解析
- 阅读小测
- 生词本

## 5. 目标用户

### 5.1 用户画像

**核心用户群**：18–26 岁大学生  
**主要使用场景**：
- 四级备考
- 六级备考
- 考研英语备考
- 日常英语阅读提升

### 5.2 典型用户类型

#### 用户类型 A：四级基础用户
- 英语基础一般，阅读速度慢
- 高频遇到生词
- 容易因为文章难度过高放弃阅读
- 更需要低门槛分级内容和便捷查词

#### 用户类型 B：六级强化用户
- 有一定词汇基础
- 阅读中主要卡在复杂句与抽象主题
- 需要更贴近考试风格的文章与理解辅助

#### 用户类型 C：考研备考用户
- 目标明确，重视阅读能力提升
- 更关注长难句分析、逻辑理解、词义辨析
- 希望在阅读中完成词汇积累与训练

### 5.3 用户痛点总结

1. 文章太难，无法坚持阅读
2. 查词频繁，打断阅读节奏
3. 看不懂长难句，影响整体理解
4. 缺乏及时反馈，不知道是否读懂
5. 阅读后没有沉淀，记不住单词与表达

## 6. 使用场景

### 场景 1：碎片时间阅读
用户在通勤、自习间隙打开 App，阅读一篇 5–10 分钟的分级文章，完成快速输入与学习。

### 场景 2：精读训练
用户在晚间系统学习时选择与考试难度匹配的文章，逐段阅读、点词查义、查看重点句解析并完成小测。

### 场景 3：备考复习
用户在考前通过生词本回顾之前阅读中积累的重点词汇，提高词汇掌握率。

## 7. 产品原则

1. **以阅读为核心**：所有功能围绕阅读体验展开，而非做成泛英语平台。
2. **学习链路闭环**：保证从文章进入到阅读完成后沉淀的完整流程。
3. **轻量可持续**：单篇文章学习时长可控，降低使用门槛。
4. **内容贴近考试**：文章难度、词汇与主题尽可能贴近四六级和考研阅读场景。
5. **优先解决核心问题**：第一版先把读懂、查词、解析、检测、积累做好。

## 10. 详细功能需求

## 10.1 首页

### 功能说明
首页用于承接用户进入 App 后的首次浏览和学习启动，帮助用户快速找到合适文章并回到上次学习进度。

### 功能点
1. 顶部搜索/推荐区域
2. 今日推荐文章卡片
3. 分级入口
4. 主题分类入口
5. 最近阅读记录
6. 生词本快捷入口

### 展示内容
- 文章标题
- 难度标签
- 考试标签（四级/六级/考研）
- 预计阅读时长
- 完成状态

### 交互要求
- 用户点击文章卡片进入阅读页
- 用户点击分级标签进入对应文章列表
- 用户点击最近阅读可回到历史进度

---

## 10.2 分级阅读模块

### 功能目标
帮助用户快速筛选适合自己英语水平与备考阶段的文章。

### 分级策略建议
产品前台展示采用“双标签”模式：

- **考试阶段标签**：四级 / 六级 / 考研
- **难度等级标签**：Level 1 / Level 2 / Level 3 / Level 4

### 难度说明
- **Level 1**：短句为主，高频词较多，适合基础弱用户
- **Level 2**：中等篇幅，出现常见学术词与常规复杂句
- **Level 3**：长难句增多，抽象主题提升
- **Level 4**：整体接近高难度考试阅读

### 列表页功能点
1. 标签筛选
2. 主题筛选（科技、教育、文化、环境、社会、心理等）
3. 排序（推荐 / 最新 / 最热）
4. 分页（Page/Size，默认 `page=1`、`size=20`，`size` 最大 50）
5. 文章卡片展示

### 卡片字段
- 标题
- 难度
- 考试阶段
- 主题
- 阅读时长
- 是否已学

### 列表接口契约（V1）
- 查询参数：`page`、`size`、`stage`、`level`、`topic`、`sort`
- 返回字段：`items`、`page`、`size`、`total`、`has_next`
- 默认排序：`recommended`

### 页面组成

#### 顶部区域

2. 用户点击重点句，查看解析

## 10.4 文章朗读

### 功能目标
辅助用户进行英语输入，提高阅读流畅性与发音感知能力。

### 功能范围（V1.0）
1. 全文朗读
2. 分段朗读
3. 播放/暂停/继续
4. 当前朗读段落高亮

### 需求说明
- 首期采用标准 TTS 即可
- 支持美式或英式音色中的一种即可
- 保证发音清晰、语速适中

### 非目标
- V1.0 暂不做跟读评分
- V1.0 暂不做变速精细控制（可后续加入）

---

## 10.5 点词释义

### 功能目标
帮助用户在不跳出阅读场景的情况下快速理解陌生词汇。

### 触发方式
用户点击正文中的单词后弹出释义卡片。

### 释义卡片字段
- 单词原形
- 音标
- 发音按钮
- 词性
- 中文释义
- 常见搭配（可选）
- 加入生词本按钮

### 需求说明
- 弹窗需轻量，不遮挡过多正文
- 用户关闭后应回到原阅读位置
- 已加入生词本的单词应有状态标识

---

## 10.6 句子解析

### 功能目标
帮助用户理解复杂句式和长难句，是本产品的重要差异化功能。

### 解析对象
- 长难句
- 从句结构
- 倒装句
- 插入语
- 非谓语结构
- 重点短语搭配

### 解析内容
1. 原句
2. 句子翻译
3. 结构拆分
4. 难点说明
5. 重点词组解释

### 触发方式
- 重点句高亮后点击进入解析
- 底部点击“句子解析”按钮查看当前段落重点句

### 需求说明
- 每篇文章至少配置 2–5 个重点解析句
- 解析内容应简洁、易懂、便于备考用户理解

### 功能范围

### 题型建议
1. 主旨题
2. 细节题
3. 词义题
4. 推理题

### 小测结果页字段

- 正确率
- 错题回顾
- 返回文章
- 下一篇推荐

### 需求说明
- 题目数量不宜过多，避免产品变成刷题工具
- 结果页应强化反馈与继续学习引导

## 11. 用户流程

### 11.1 核心学习流程

1. 用户进入首页
2. 选择考试阶段与难度
3. 进入文章列表
4. 点击文章进入阅读页
5. 在阅读过程中使用朗读、点词释义、句子解析
6. 完成阅读后进入小测
7. 查看结果并进入下一篇或返回首页
8. 阅读中收藏的生词进入生词本

### 11.2 生词积累流程

1. 用户阅读时点击生词
2. 查看释义卡片
3. 点击加入生词本
4. 进入生词本查看并管理状态

### 11.3 回访流程

1. 用户再次打开 App
2. 首页展示最近学习文章与推荐内容
3. 用户继续上次阅读或开始新文章

## 12. 业务规则

1. 每篇文章需绑定一个考试阶段标签与一个难度等级标签。
2. 每篇文章至少包含一段朗读音频或 TTS 能力。
3. 每篇文章至少配置若干可点击释义的词汇。
4. 每篇文章至少配置 2–5 条重点句解析。
5. 每篇文章至少配置 3 道小测题目。
6. 生词本允许同一单词来自不同文章，数据层按“用户 + 单词 + 来源文章”存储，前端默认按单词去重展示。
7. 生词本需支持按文章来源筛选来源明细。
8. 已完成小测的文章应记录完成状态。
9. 收藏文章纳入 V1：同一用户对同一文章仅保留一条有效收藏记录，重复收藏需幂等。
10. 收藏记录需包含状态与时间戳（收藏时间、取消收藏时间可选），并可在“我的-收藏文章”中查询。
11. 收藏状态与阅读历史互不覆盖：取消收藏不删除阅读历史，阅读完成不自动收藏。

## 15. 非功能需求

### 15.1 性能要求
- 首页加载时间控制在 2 秒内
- 阅读页打开时间控制在 2 秒内
- 点词释义弹窗响应时间 < 500ms
- 音频播放启动时间 < 1 秒

### 15.2 可用性要求
- 阅读页字体清晰，支持适中的字号展示
- 释义弹窗不影响主要阅读体验
- 页面跳转层级不宜过深

### 15.3 内容要求
- 文章内容应符合大学生英语学习场景

## 19. 风险与注意事项

1. **内容质量风险**：如果文章、释义、解析和小测质量不高，产品价值会显著下降。
2. **分级不准确风险**：文章分级过难或过易，会影响用户留存。
3. **功能分散风险**：若过早加入过多学习模块，易削弱阅读核心体验。
4. **工具化不足风险**：若点词、解析和小测体验不顺畅，用户会认为只是普通阅读器。
5. **技术投入失衡风险**：前期应优先保障内容闭环，而非投入过多在复杂算法或社交功能上。

## 20. 验收标准

### 20.1 产品验收
- Windows 客户端可安装并完成首次启动
- 用户可完成从首页到阅读到小测到生词收藏的完整流程
- 不同难度文章可正常筛选与展示
- 文章列表支持分页与排序，翻页结果稳定
- 阅读页可正常播放音频、点词查看释义、查看解析
- 生词本可正常新增、查看、标记状态
- 小测提交后可正常展示结果
- 收藏文章可新增、取消、查询，且与阅读历史互不干扰

### 20.2 体验验收
- 用户能在 1 分钟内找到并开始阅读一篇文章
- 用户在阅读中无需离开主页面即可完成查词
- 用户可明显感知“读懂了”和“积累了”的反馈
- 鼠标与键盘基础操作（滚动、点击、返回）在阅读主链路中可用

### 20.3 接口契约验收样例（文档级）
1. `GET /api/v1/articles?page=1&size=20&stage=cet4&level=2&topic=education&sort=recommended`
   - 期望返回：`items/page/size/total/has_next` 字段齐全，`size` 不超过 50。
2. `POST /api/v1/vocab`
   - 期望同一 `word_id` 在不同 `source_article_id` 下可分别入库；生词列表默认按词去重展示，可按 `source_article_id` 查看来源明细。
3. `POST /api/v1/articles/{article_id}/favorite` 与 `DELETE /api/v1/articles/{article_id}/favorite`
   - 期望重复收藏幂等；取消后可再次收藏；收藏状态不影响阅读历史。
4. `GET /api/v1/articles/{article_id}/audio`
   - 期望返回音频状态：`pending` / `processing` / `ready` / `failed`，`failed` 时客户端降级为文本阅读。
