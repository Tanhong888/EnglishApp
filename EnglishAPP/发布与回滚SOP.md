# EnglishAPP 发布与回滚 SOP（V1）

## 1. 目标

将 `TECH_DESIGN.md` 中的 `staging -> prod` 发布流程落成可执行清单，确保：

- 发布前有统一检查口径
- 发布后有可观察验证步骤
- 失败时有明确回滚动作

---

## 2. 环境说明

- `dev`：本地开发联调
- `staging`：预发布验证
- `prod`：生产环境

---

## 3. 发布前检查

发布前必须完成：

1. 后端测试通过

```powershell
cd backend
python -m pytest tests -q
```

2. 桌面端静态检查与测试通过

```powershell
cd desktop_app
flutter analyze
flutter test
```

3. 关键人工回归通过

- 登录/登出
- 首页推荐
- 文章列表筛选与分页
- 阅读详情
- 句子解析
- 小测提交与结果页
- 生词本
- 我的页学习记录

4. 本次发布需附变更说明

- 新功能
- 修复项
- 已知风险

---

## 4. Staging 发布步骤

1. 合并代码到主分支
2. CI 自动执行后端与桌面端检查
3. 部署到 `staging`
4. 执行一次完整主链路冒烟：

- 登录
- 打开文章详情
- 点词
- 查看句子解析
- 完成小测
- 查看学习记录

5. 记录结果并确认是否进入生产发布

---

## 5. 生产发布步骤

1. 确认 `staging` 冒烟通过
2. 备份生产数据库
3. 发布生产版本
4. 发布后 10 分钟内重点验证：

- `/health`
- `/docs`
- 登录接口
- 文章列表接口
- 阅读详情接口
- 小测提交接口

5. 观察错误日志、接口错误率、响应时间

---

## 6. 回滚触发条件

满足任一条件立即回滚：

- 登录主链路不可用
- 阅读详情无法打开
- 小测提交失败率明显升高
- 生产错误日志持续增加
- 发布后关键接口出现 5xx

---

## 7. 回滚步骤

1. 停止继续放量
2. 回退到上一个稳定版本
3. 如有数据库变更，按迁移脚本执行 `downgrade` 或恢复备份
4. 重新验证以下能力：

- 登录
- 首页
- 阅读详情
- 小测
- 生词本

5. 记录事故时间、影响范围、原因与修复计划

---
