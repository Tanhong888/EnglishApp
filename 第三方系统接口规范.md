# EnglishAPP 第三方系统接口规范

## 1. 文档目的

本文档用于规范 EnglishAPP 与第三方系统的集成方式，便于后续对接：

- TTS 语音服务
- 外部词典服务
- 对象存储
- 缓存/队列基础设施
- 告警与可观测系统

说明：本文档同时包含“已接入”和“预留方案”两类内容，并明确状态。

---

## 2. 集成清单与状态

| 系统 | 作用 | 当前状态 |
|---|---|---|
| PostgreSQL | 业务主库 | 已支持（可通过 `DATABASE_URL` 配置） |
| Redis | 缓存/异步扩展预留 | 已支持基础连接配置 |
| 云 TTS 服务 | 文章音频生成 | 预留（V1 仅状态字段与占位返回） |
| 外部词典 API | 补充词条数据 | 预留（当前主要使用本地 `words` 表） |
| 对象存储（S3/OSS/COS） | 音频文件存储 | 预留 |
| 告警/监控（Sentry/Webhook） | 异常上报与告警 | 预留 |

---

## 3. 通用对接约束

### 3.1 协议与编码

- 协议：HTTPS
- 编码：UTF-8
- 数据格式：JSON（除音频二进制上传/下载）

### 3.2 鉴权与密钥

- 第三方密钥统一放入环境变量或密钥管理系统
- 禁止硬编码在仓库
- 本地 `.env` 与生产配置必须分离

### 3.3 超时与重试

- 默认超时：
- 连接超时：2-3 秒
- 读超时：5-10 秒

- 重试策略：
- 仅对幂等请求重试
- 最多 3 次，指数退避（例如 1s/2s/4s）

### 3.4 可观测性

- 每次第三方调用记录：
- 目标系统
- 请求耗时
- 状态码
- trace_id

- 禁止记录敏感明文（例如密钥、完整 token）

## 4. TTS 服务接口规范（预留）

### 4.1 目标

将文章文本转换为音频文件，并支持状态机：

- `pending -> processing -> ready | failed`

### 4.2 统一请求体（建议）

```json
{
  "task_id": "article_123_full",
  "text": "...",
  "lang": "en-US",
  "voice": "default_male",
  "speed": 1.0,
  "format": "mp3"
}
```

### 4.3 统一响应体（建议）

```json
{
  "provider_task_id": "abc123",
  "status": "accepted",
  "audio_url": null,
  "error_code": null,
  "error_message": null
}
```

### 4.4 回调规范（建议）

第三方回调到我方服务时，至少包含：

```json
{
  "provider_task_id": "abc123",
  "status": "ready",
  "audio_url": "https://.../audio.mp3",
  "duration": 123.4,
  "checksum": "..."
}
```

### 4.5 失败处理

- 失败重试最多 3 次
- 超过重试上限标记 `failed`
- 触发告警（邮件/IM/Webhook）

## 5. 外部词典接口规范（预留）

### 5.1 使用场景

当本地 `words` 表未命中词条时，可调用外部词典补全，再选择是否回写本地库。

### 5.2 请求规范（建议）

- Method: `GET`
- Path: `/lookup`
- Query: `word=<lemma>&lang=en`

### 5.3 响应规范（建议）

```json
{
  "word": "consolidate",
  "phonetic": "kənˈsɑːlɪdeɪt",
  "pos": "vt.",
  "definitions": ["巩固"],
  "examples": ["..."],
  "source": "provider_name"
}
```

### 5.4 字段映射到本项目

- `word` -> `words.lemma`
- `phonetic` -> `words.phonetic`
- `pos` -> `words.pos`
- `definitions[0]` -> `words.meaning_cn`

## 6. 对象存储接口规范（预留）

### 6.1 文件命名建议

- `audio/articles/{article_id}/full.mp3`
- `audio/articles/{article_id}/segments/{index}.mp3`

### 6.2 上传策略

- 优先使用服务端签发的预签名 URL
- 客户端/Worker 通过预签名 URL 上传
- 上传后回写业务库 `article_audio_url`

---

## 7. Redis / 队列规范（基础可用）

### 7.1 连接

- 通过 `REDIS_URL` 配置
- 建议实例级限流与连接池配置

### 7.2 建议用途

- 热点查询缓存（首页推荐/词条）
- TTS 异步任务队列（后续）
- 登录/查词等接口限流计数（后续）

---

## 8. 错误码与降级约定

### 8.1 第三方错误分层

- 网络错误：超时、DNS、连接失败
- 协议错误：4xx/5xx
- 业务错误：参数非法、配额不足、鉴权失败

### 8.2 降级策略

- TTS 不可用：返回文本阅读能力，音频按钮降级
- 词典不可用：优先本地缓存词条，提示稍后重试
- 对象存储不可用：保留任务状态并触发重试

---

## 9. 安全与合规要求

- 全链路 HTTPS
- 密钥轮换周期建议 <= 90 天
- 不在日志中打印敏感信息
- 第三方回调需要签名校验（建议 HMAC）
- 生产环境对外调用必须有超时、重试、熔断保护

---

## 10. 验收清单

1. 配置项已落地到环境变量（非硬编码）
2. 第三方调用具备超时与重试
3. 调用日志可追踪（含 trace_id）
4. 失败场景具备降级与告警
5. 回调接口具备签名校验与幂等处理
